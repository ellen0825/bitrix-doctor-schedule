# Тестовое задание: управление расписанием врача

## Что сделано

Реализован модуль **vit.doctor.schedule** для продукта «Персональный сайт врача» (vit.doctor) на 1С-Битрикс.

- **Интерфейс в Bitrix:** страница настройки рабочей недели (Настройки → Расписание врача или `/bitrix/admin/vit_doctor_schedule.php`). Для каждого дня недели: чекбокс «Рабочий день» и поля «Время начала» / «Время окончания». Пример: Понедельник 09:00–18:00, Вторник 09:00–18:00, Среда — выходной.
- **Сохранение:** по кнопке «Сохранить» данные записываются в БД.

## Краткое описание архитектуры решения

**Хранение:** отдельная таблица БД `b_vit_doctor_schedule`. Одна строка = один день недели для одного врача. Поля: `DOCTOR_ID` (0 — по умолчанию), `WEEKDAY` (MON–SUN), `IS_WORKING` (Y/N), `TIME_FROM`, `TIME_TO`. Уникальный ключ `(DOCTOR_ID, WEEKDAY)` — без дублей, удобно масштабировать на нескольких врачей.

**Доступ:** D7 Entity `ScheduleTable` и сервисный класс `Schedule`: `getWeekSchedule($doctorId)` и `saveWeekSchedule($data, $doctorId)`. Админка использует `$doctorId = 0`.

**Установка:** при установке модуля выполняется `install/db/mysql/install.sql` (создание таблицы), заполняется шаблон Пн–Пт 09:00–18:00. При удалении — `uninstall.sql` (DROP TABLE). Файлы админки копируются в `bitrix/admin/`, в меню добавляется пункт «Расписание врача».

**Почему таблица, а не опции/JSON:** нормализованные данные, один формат для любого числа врачей, привычная модель Bitrix (Entity + install/uninstall), возможность позже добавить исключения по датам без смены структуры.
