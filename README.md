# Модуль «Расписание врача» (vit.doctor.schedule)

Модуль для решения [Персональный сайт врача (vit.doctor)](https://marketplace.1c-bitrix.ru/solutions/vit.doctor/). Добавляет в админку Bitrix настройку **рабочей недели врача**: рабочие дни и время начала/окончания работы.

## Требования

- 1С-Битрикс: Управление сайтом
- PHP 7.4+

## Установка

1. Скопируйте папку `vit.doctor.schedule` в каталог модулей:
   - **local/modules/vit.doctor.schedule** (предпочтительно)
   - либо **bitrix/modules/vit.doctor.schedule**

2. В админке Bitrix: **Настройки → Настройки продукта → Модули** — найдите «Расписание врача (vit.doctor)» и нажмите **Установить**.

3. После установки откройте страницу настройки расписания:
   - **https://ваш-сайт.ru/bitrix/admin/vit_doctor_schedule.php**

## Использование

На странице настройки задаётся для каждого дня недели:

- **Рабочий день** — включён/выключен (например, среда — выходной).
- **Время начала** и **время окончания** работы (например, 09:00–18:00).

Пример: понедельник и вторник 09:00–18:00, среда выходной, четверг и пятница 09:00–18:00, суббота и воскресенье выходные.

Данные сохраняются по кнопке **Сохранить**.

## Использование в коде (форма записи)

Расписание доступно через класс `\Vit\DoctorSchedule\Schedule`:

```php
use Vit\DoctorSchedule\Schedule;

// Получить расписание на неделю
$schedule = Schedule::getWeekSchedule();
// $schedule['MON'] = ['is_working' => 'Y', 'from' => '09:00', 'to' => '18:00']
// $schedule['WED'] = ['is_working' => 'N', 'from' => '09:00', 'to' => '18:00']
// ...
```

По этому массиву можно строить доступные слоты для записи (например, разбивать интервал `from`–`to` на 30-минутные окна и отдавать на фронт).

## Краткое описание архитектуры решения

### Хранение данных

- Расписание хранится в **опциях модуля** Bitrix (`\Bitrix\Main\Config\Option`): одна опция `doctor_week_schedule`, значение — JSON.
- Структура: по одному объекту на день недели (MON–SUN) с полями `is_working`, `from`, `to`. Для выходного дня достаточно `is_working = N`, время не используется, но сохраняется для единообразия.
- **Почему так:** минимум зависимостей, не нужны миграции и таблицы; при необходимости легко расширить до нескольких врачей (например, ключ по `doctor_id` или отдельная опция на врача).

### Админ-интерфейс

- Отдельная страница в `bitrix/admin/` (файл копируется при установке модуля). Форма: таблица из 7 строк (дни недели), для каждого дня — чекбокс «Рабочий день» и два поля времени (начало/конец). При сохранении данные нормализуются и записываются в опцию через `Schedule::saveWeekSchedule()`.

### Связь с фронтом

- Текущая реализация модуля записи в vit.doctor не меняется. Модуль предоставляет только данные: `Schedule::getWeekSchedule()`. Интеграция с формой записи: в компоненте или в обработчике записи вызывать `getWeekSchedule()`, по нему формировать список доступных слотов (день + время) и подставлять в существующую вёрстку/логику.

### Расширение

- Для нескольких врачей: в опции хранить массив по `doctor_id` или завести Highload-блок/таблицу с полями `doctor_id`, `weekday`, `is_working`, `time_from`, `time_to` и перенести туда логику чтения/записи; API `getWeekSchedule($doctorId)` оставить тем же по смыслу.
